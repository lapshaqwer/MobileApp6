## Контрольные вопросы:
1. Какие уровни логирования существуют в Android? Для каких целей используется каждый из них? Verbose (v) — всё подряд, Debug (d) — отладка, Info (i) — информирование, Warn (w) — предупреждения, Error (e) — ошибки.

2. Как открыть окно Logcat в Android Studio? Как отфильтровать сообщения только по тегу и только по уровню Error? Logcat: Alt+6 (Win/Linux); фильтр по тегу — tag:мой_тег; по уровню Error — выбрать Error в выпадающем списке уровней.

3. В чем разница между методами Log.e() и Log.w()? Приведите примеры использования. Log.e() — ошибка (красный), Log.w() — предупреждение (оранжевый); пример: Log.e("TAG", "Ошибка парсинга") vs Log.w("TAG", "Поле пустое").

4. Что такое точка останова (breakpoint)? Как запустить приложение в режиме отладки? Точка останова — место остановки кода для отладки; запуск в отладке — кнопка «жучок» или Shift+F9.

5. Как выполнить код с задержкой в Android? Назовите не менее двух способов. Два способа: timer.schedule(TimerTask task, long delay)  и timer.schedule(TimerTask task, long delay, long period).

6. В чем проблема обновления UI из задачи, выполняемой в TimerTask? Как её решить? TimerTask в фоновом потоке, UI трогать нельзя; решение — runOnUiThread { ... }.

7. Для чего используется класс Chronometer? Какие основные методы у него есть? Chronometer — виджет-секундомер; основные методы: start(), stop(), setBase(), setFormat().

8. Чем CountDownTimer отличается от Timer? В каких случаях удобнее использовать CountDownTimer? CountDownTimer — обратный отсчёт (тики в UI-потоке), Timer — периодические задачи (фон); CountDownTimer удобен для таймера на экране (например, "осталось 30 секунд").
